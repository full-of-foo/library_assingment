1) Static Root Page and Sessionless Content

Use Case

As a visitor to 'crappy-book-store.com', a book store service, I want to see the typical sessionless conent of a service-based website. I want to see a root page where I see a summary of the service offering and links for further information.

I want to see an 'about' navigation link, which will display further information on the offering. I want to see a 'help' navigation link, which will display helpful information around the offering. I want to see a 'sign up' navigation link, which will display a sign up form.

User Stories

As a visitor to the site

I navigate to "/"

I see the "h1" with with the text "Crappy-Book-Store.com"

I see the "a" with with the text "Sign in"

I see the site's static top navigation bar items (about, help, sign up)

I topbar navigate to the about page

I see the about page

I topbar navigate to the help page

I see the help page

I topbar navigate to the sign up page

I see the sign up page

2) Sign Up/Out Customer

Use Case

As a visitor, I want to sign up to this service. While on the root page, I click a link to sign up

I want to see a sign up 'form', which will requires my user details. After sign up I want to be automatically logged in and redirected to '/home'. On the '/home' page, I want to see a 'sign out' navigation link, which signs me out and redirects me to '/login'.

User Stories

As a visitor to the site

I navigate to "/"

I see sign up and login links

I click sign up and I see a sign up form

I submit invalid details and validations are returned

I submit correct details

I see a welcome banner and my profile page

I see the top navbar links for 'books', 'profile', 'home' and 'sign out'

I click the sign out link and I see that I am redirected to sessionless pages

3) Basic Customer Cruds

Use Case

As a customer, I expect to be able to update my profile details and also be able to delete my account.

User Stories

As a logged in customer

I navigate to my profile

I see an edit button and a delete account button

I click the edit button

I see an edit form and I can update my account

I return to my profile and click delete, then this deletes my account

I then see the sessionless home page

4) Admin creation

Use Case

As an admin, I can create other admins, not update or delete them.

As a customer, I should not have permission to browse/list other customers or admins.

User Stories

As a logged in admin

I navigate to the users tab

I see an 'enroll admin' button on this 'users' page

I can create fellow admins

As a logged in customer

I attempt to navigate to "/users", I get redirected error

6) Book Show, List and Filter

Use Case

As a customer or admin, I expect to be able to topbar navgate to 'Books'

I expect to be able to be able to see to book columns 'title', 'author', 'topic', and 'price'

I expect to be able to sort by these columns

I expect to be able to search by 'author' and 'title'

User Stories

As a known customer or admin

I topbar navigate to 'Books'

I see columns 'title', 'author', 'topic', and 'price'

I can sort the book list by these columns

I can search for books via title or author

I can open a book and see the book show page
